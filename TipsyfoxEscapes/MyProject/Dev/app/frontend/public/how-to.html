How To Use

Follow the Flow strip at the top of the builder. Steps match what you see in the app.

1. Sign in

  1. Create an account or log in. The builder assumes a signed-in user for saving and for catalog limits.

2. Room details

  1. Enter players at one time, total participants, and session duration (minutes).
  2. Enter environment (where the game runs) and available items as tags — the server uses these for themes, puzzle hooks, and export ideas. See How available items become puzzles below.
  3. Set room difficulty and optional junior add-on options if you need a parallel easier track.
  4. Click Continue to themes (or equivalent). Incomplete required fields are highlighted.

3. Choose a theme

  1. On the free trial, the same three curated themes load automatically when you reach this step — pick a theme with the radio, then Continue to puzzle builder. With a paid room pack / full catalog, use Generate For Me or Refresh Ideas for the wider rotating set.
  2. Use My Own Theme — only if you have full catalog access (room pack). Enter a theme name.
  3. Write or paste your theme description in the large text area above the theme coach (markdown-friendly sections are fine).
  4. Optional: use the theme coach below that field when Chrome on-device AI is enabled; use Apply answers to description to merge interview answers into the text area.
  5. Click Add and Select Custom Theme to save the custom theme and jump to Build puzzle set. If something fails, read the red banner at the top of the Flow card.

How available items become puzzles

  • Each tag is a physical anchor. List one prop per tag (sofa, ping-pong table, TV, side table). The generator folds them into the theme brief as a numbered story-integration list: where the object sits in the fiction and how a puzzle uses it (clues on a table, a looping video on the TV, a lockbox on a side table, tags in sofa cushions, and similar).
  • Puzzles and copy reference those anchors. Generated puzzles get inventory tie-ins, and the export includes a Build team anchor checklist so ops knows which objects are load-bearing for logic — not movable without redesign.
  • “Add existing puzzle” links are stored as references in your plan and export (the app does not scrape them). Puzzle reference links after generation are for technique, credit, and safety — adapt into original venue builds.
  • Polish with on-device AI. On the theme step, use Editor pass (when available in supported browsers) so awkward generator sentences read more cleanly — without changing your inventory contract.

4. Build puzzle set

  1. Optionally add existing puzzles you already own (name, link, room part).
  2. Click Generate puzzles when you are ready. Puzzle count follows the room timing and difficulty you saved earlier.
  3. Use Continue to output review after you have at least one generated puzzle (the button stays disabled until then). The Output: Review card opens below the Flow summary; the view scrolls to it. If room details are invalid, fix them via ← Back to Room details—the app shows a red banner at the top of the Flow card.

5. Output

  1. Review the puzzle list and storyline. Use Continue to Export when ready.
  2. Run Export Plan, optionally Mark Approved for Build, then Save Plan to keep it on your account.

Local development

Run the backend from Dev/app/backend and the frontend from Dev/app/frontend (see project README / AGENTS). Open the Vite dev URL (not the raw index.html file) so scripts load correctly.
