Builder workflow

1. Sign in

  1. Create an account or log in. The builder assumes a signed-in user for saving and for catalog limits.

2. Room details

  1. Enter players at one time, total participants, and session duration (minutes).
  2. Enter environment (where the game runs) and available items as tags — the server uses these for themes, puzzle hooks, and export ideas. See How available items become puzzles below.
  3. Set room difficulty and optional junior add-on options if you need a parallel easier track.
  4. Choose Target interface: Home Party optimizes print runbooks; Commercial Venue unlocks the Gamemaster Live Console, staff checklists, and multi-display ops.
  5. Click Continue to themes (or equivalent). Incomplete required fields are highlighted.

3. Choose a theme

  1. On the free trial, the same three curated themes load automatically when you reach this step — pick a theme with the radio, then Continue to puzzle builder. With a paid room pack / full catalog, use Generate For Me or Refresh Ideas for the wider rotating set.
  2. Use My Own Theme — only if you have full catalog access (room pack). Enter a theme name.
  3. Write or paste your theme description in the large text area above the theme coach (markdown-friendly sections are fine).
  4. Optional: use the theme coach below that field when Chrome on-device AI is enabled; use Apply answers to description to merge interview answers into the text area.
  5. Click Add and Select Custom Theme to save the custom theme and jump to Build puzzle set. If something fails, read the red banner at the top of the Flow card.

Physical anchor checklist (logic-aware AI)

The generator is logic-aware: it treats each Available items tag as a real prop in your room, not decorative flavor text.

  1. Name props precisely. Use beaker, microscope, whiteboard — not “science stuff.” The export’s Narrative justification for each puzzle will cite where that object lives and why the clue belongs there.
  2. One tag = one object. Split combined lists into separate chips so the AI can assign distinct puzzle hooks per prop.
  3. Match your environment field. Set Environment to the fiction zone (e.g. “Science Lab”) so “where” lines in the export read like a build map, not a generic puzzle blurb.
  4. Review the anchor checklist in export. After export, open the Build Team Anchor Checklist — those props are contractually tied to puzzle logic; moving them requires redesign.

Export documentation (build-ready PDF)

Exported plans are formatted for professional build teams, not vague brainstorm notes. After Export plan, download .html and use your browser’s Print → Save as PDF.

  • Consolidated bill of materials table at the front (components, sensors, props per puzzle).
  • Narrative justification on every puzzle — explicit why and where (e.g. “Inside the Science Lab, the player finds a beaker containing…”).
  • Build resources with real links — Playful Technology for electronics, plus curated supplier/build guides (no placeholder code).
  • Master blueprint, Mermaid flowchart with your puzzle names, and Live Ops — Gamemaster briefing (clue triggers, progression, failure recovery).

Old format: vague logic

Generic bullet list of puzzle titles with one-line objectives and no prop placement or procurement table.

New format: build-ready specs

Per-puzzle narrative justification, BOM rows, wiring sequences, GM clue triggers, and printable page breaks between phases.

How available items become puzzles

  • Each tag is a physical anchor. List one prop per tag (sofa, ping-pong table, TV, side table). The generator folds them into the theme brief as a numbered story-integration list: where the object sits in the fiction and how a puzzle uses it (clues on a table, a looping video on the TV, a lockbox on a side table, tags in sofa cushions, and similar).
  • Puzzles and copy reference those anchors. Generated puzzles get inventory tie-ins, and the export includes a Build team anchor checklist so ops knows which objects are load-bearing for logic — not movable without redesign.
  • “Add existing puzzle” links are stored as references in your plan and export (the app does not scrape them). Puzzle reference links after generation are for technique, credit, and safety — adapt into original venue builds.
  • Polish with on-device AI. On the theme step, use Editor pass (when available in supported browsers) so awkward generator sentences read more cleanly — without changing your inventory contract.

4. Build puzzle set

  1. Optionally add existing puzzles you already own (name, link, room part).
  2. Click Generate puzzles when you are ready. Puzzle count follows the room timing and difficulty you saved earlier.
  3. Use Continue to output review after you have at least one generated puzzle (the button stays disabled until then). The Output: Review card opens below the Flow summary; the view scrolls to it. If room details are invalid, fix them via ← Back to Room details—the app shows a red banner at the top of the Flow card.
  4. Reject a single puzzle with Replace — only that slot regenerates; your accepted puzzles stay put.

5. Output

  1. Review the puzzle list and storyline. Use Continue to Export when ready.
  2. Run Export Plan, optionally Mark Approved for Build, then Save Plan to keep it on your account.
  3. On the Export step, use Run your game to open live URLs (commercial venue) or the player screen alone (home party).

Builder → Live Console workflow

Bridge the build packet (PDF/HTML export) with game night performance (live URLs).

  1. Finish Room details → Theme → Generate puzzles → Export plan (download .md or .html for your build crew).
  2. Keep the same session id — it links the saved plan, export, GM console, and player display.
  3. On the Export step, open Gamemaster Live Console and Launch player screen (commercial venue) or player screen only (home party).
  4. Run the game from the export’s Live Ops — Gamemaster briefing table (clue triggers and failure recovery) while driving the console in real time.

See Live operations for URLs, clue box behavior, and venue layout.

Live operations

Use these after you export a plan. URLs use your planning session id (shown in the builder and GM console header). Replace {sessionId} with yours.

Gamemaster Live Console (master controller)

The GM console is the only master for timer, stage, puzzle completion, and clue delivery. Open it on a crew tablet or laptop that stays with your game master — not on the projector.

  • URL: /gm/{sessionId} (alias: /console/{sessionId})
  • Access: Commercial venue / operator subscription with GM console enabled. If subscription is inactive, the console is read-only and clues are frozen.
  • Console tab: Start or pause the countdown, adjust ±1 minute, set active player count, advance story stage, mark puzzles complete, run the reset checklist, and end the game (success or fail).
  • Screen Manager tab: Push player display modes — Active Game, Hint Overlay, End Game, or Custom Media Trigger (with your label for branded slides).
  • Reports tab: Session result, elapsed time, bottleneck puzzle, and recent event log.

Player window

Open this on the in-room projector, TV, or wall-mounted tablet so teams see the timer and hints at a glance.

  • URL: /room/{sessionId}/player-display
  • Shows plan name, live timer, puzzle progress, and the current hint when the GM sends one.
  • Wait for Player display ready on the GM console before you start the clock — that means the player page has connected to the live stream.
  • Home party: You can run player-only with a printed runbook on your phone; commercial venues pair both URLs.

Clue box — real-time sync

On the GM console Player window tab (or the main flow when clues are in context), use the GM live clue box:

  1. Type a custom clue and press Send (or Enter).
  2. Or tap a pre-saved hint chip from your exported plan — faster for repeat nudges.
  3. The player display updates over the live stream; sending a clue often switches the projector to Hint Overlay so the hint is prominent.
  4. Clues respect your operating mode policy (venue vs home); invalid or empty clues are rejected with a message on the console.

Venue ops — displays & physical layout

Standard two-display layout (also summarized in the console’s Venue ops help disclosure):

  • Display 1 (GM): Crew device → /gm/{sessionId} — timer, clue box, puzzle log, screen modes.
  • Display 2 (players): Projector or tablet → /room/{sessionId}/player-display — large timer + hints only; no GM controls exposed.
  • Layout: Teams face Display 2; GM works from the side or back of the room so they can see both the floor and their tablet. Tape HDMI/USB runs; label which cable is “player” vs “GM” for teardown.
  • Multi-room fleets: Each room needs its own session id and display pair; enterprise provisioning is required for fleet features (see Venue Blueprint onboarding if the console redirects you).

Quick start from Export

  1. Export the plan and note your session id.
  2. On the Export step, click Open Gamemaster Live Console (venue) and Launch player screen.
  3. Open the player URL on the projector; open the GM URL on your crew device.
  4. Confirm Player display ready, then start the timer and run the game.

Admin (operators)

Internal tool for staff — not part of the host-facing game night flow.

  • URL: /admin/dashboard
  • Requires: Admin role and signed-in bearer token (same auth as the builder).
  • Use for: Search users, adjust room allowance and export credits, set lifecycle/subscription flags, toggle enterprise provisioning, and review the audit log.

Local development
