Chapter 3

Generating & Refining the Puzzle Set

Once a theme is selected, generate a complete puzzle set. The system aims for variety across logic, physical, and electronic categories while respecting your room inputs.

Run Generation

  1. Confirm the selected theme and planning summary.
  2. Add existing puzzle references only when they should influence the new set.
  3. Run Generate puzzles. Puzzle count scales from duration, concurrent players, difficulty, and any junior add-on configuration.

Replace One Weak Card

Use Replace on a single puzzle card when that idea is not buildable or does not fit the space. Replacement updates only that slot; accepted puzzles remain unchanged so the plan does not churn unnecessarily.

Use On-Device Editorial Passes

When supported in the browser, the optional on-device AI pass polishes wording such as how it works, theme fit reason, stages, and suggested additions. It does not need to change your physical inventory contract.

Approval standard: every accepted puzzle should have a clear objective, a solve path, a physical or digital anchor, and enough construction detail for the export packet to be useful to builders.
